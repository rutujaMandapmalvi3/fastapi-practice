There is no built-in way to check programmatically whether a named route exists in a FastAPI application without iterating over all routes manually. Add a method to the FastAPI application class that returns whether a route with a given name or path is registered.

Learnings:
- Code must live inside the FastAPI class in fastapi/applications.py as a new method.
- The iteration pattern to reference is inside fastapi/routing.py inside include_router(), which loops over router.routes (a list of BaseRoute objects) and reads each route's name attribute.
- Incoming requests go through FastAPI.__call__ (applications.py:1194) but everything is actually handled by Starlette — __call__ is not the right place.
- The new method takes a name (str) as input, scans self.router.routes, and returns True/False depending on whether a route with that name exists.
- Insert the method at applications.py:1198, between __call__ and add_api_route.
- self.router is the APIRouter object (created at applications.py:1003). The actual list of all registered routes is self.router.routes — that is the list to scan in the new method.

Method to implement:

def route_exists(self, name: str) -> bool:
    for route in self.router.routes:
        if route.name == name:
            return True
    return False
