FastAPI allows hiding individual routes from the OpenAPI schema using include_in_schema on route 
decorators, but there is no way to hide all routes under an entire router at once. Add support for 
controlling OpenAPI schema inclusion at the router level so that all routes registered under a router 
can be excluded without modifying each route individually.


logic for hiding individual routes OpenAPI schema exists - uses 2 flags - route's and router's - include_in_schema - route 
decorators

main task - cant hide all routes  under an entire router at once - add support for controlling openAPI 
schema inclusion at router level 

Findings:

Entry point is applications.py. Hiding logic lives in routing.py and openapi/utils.py.
include_in_schema is the flag controlling visibility in the OpenAPI schema. It does NOT affect actual request handling — hidden routes still work, they just don't appear in /docs.
The flag is ANDed at route registration time: route flag AND router flag AND include_router() flag → baked into each APIRoute object.
app.routes is a flat list of APIRoute objects. After include_router() runs, routes are copied as new objects — no link back to the source router is preserved.
Options considered:

Match routes by endpoint function reference (same function object survives the copy)
Use path prefix to identify routes belonging to a router
Stamp each copied route with its source router during include_router()
Solution:

Add a router_source attribute to APIRoute objects, stamped during include_router() at routing.py:1764
Add a new method that takes a router, filters app.routes by router_source, and sets include_in_schema on matched routes
Only one risky location: include_router() reconstruction at line 1764 — must pass router_source through add_api_route() or it gets silently dropped
WebSocket routes don't have include_in_schema — skip them
All OpenAPI generation code in openapi/utils.py is read-only — unaffected
